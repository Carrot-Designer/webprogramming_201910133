## 1. 프롬프트 품질 분석

### 1.1 강점 ✅

#### 명확한 목표 설정
```
"모든 게시물의 제목에 맞는 detail page를 만들어줘"
```
- 원하는 결과물이 명확함
- 작업 범위가 구체적으로 정의됨

#### 한국어-영어 혼용의 유연성
- 기술 용어는 영어로, 맥락 설명은 한국어로 적절히 혼용
- AI가 이해하기 쉬운 형태로 전달

### 1.2 개선 필요 영역 ⚠️

#### 모호한 요구사항
```
"section-header를 삭제하고 best-posts reveal active를 대신 삽입해줘"
```
**문제점**:
- "대신 삽입"의 정확한 위치 불명확
- 기존 구조를 유지할지, 대체할지 모호함
- 예상 결과물에 대한 설명 부재

**권장 프롬프트**:
```
"my page의 Step 5 섹션에서:
1. <div class="section-header"> 전체를 삭제
2. <section class="best-posts reveal">을 <section class="best-posts reveal active">로 변경
3. 나머지 구조는 유지"
```

#### 컨텍스트 부족
```
"아직 클릭해도 페이지가 안나오는 게시물이 많아"
```
**문제점**:
- 어떤 페이지에서 문제가 발생하는지 불명확
- 몇 개의 게시물이 문제인지 수치 없음
- 에러 메시지나 증상 설명 부재

**권장 프롬프트**:
```
"postlist page에서 6개의 정적 카드 중 5개가 클릭 시 404 에러 발생.
현재 onclick이 '../postdetail page/index.html'로 되어 있는데,
각 제목에 맞는 detail page 경로로 수정 필요"
```

### 1.3 프롬프트 패턴 분석

| 패턴 | 빈도 | 효과성 |
|------|------|--------|
| 단순 명령형 | 높음 | 중간 |

## 2. 프로젝트 관리 역량

### 2.1 강점 ✅

#### 빠른 피드백 루프
- 결과물 확인 후 즉시 수정 요청
- 문제 발견 시 바로 보고

#### 점진적 개선 의지
- 완벽을 기대하지 않고 반복 수정 수용
- "아직 안 되는 부분이 있다"고 솔직하게 전달

### 2.2 개선 필요 영역 ⚠️

#### 사전 검증 부족
**문제 상황**: 링크가 작동하지 않는 문제가 여러 차례 반복 발생

**원인 분석**:
- 작업 완료 후 직접 테스트하지 않고 다음 요청 진행
- 부분적 확인 후 전체가 완료되었다고 가정

**권장 프로세스**:
```
1. AI 작업 완료 → 2. 직접 브라우저에서 테스트 
→ 3. 문제 발견 시 구체적 증상과 함께 보고
→ 4. 전체 동작 확인 후 다음 작업 진행
```

#### 요구사항 분할 미흡
**문제 상황**: 
```
"1. section-header를 삭제하고... 2. 모든 페이지에서 연결해줘"
```

**문제점**:
- 두 가지 다른 성격의 작업을 하나의 요청에 혼합
- 첫 번째 작업의 완료 확인 없이 두 번째 작업 동시 요청

**권장 방식**:
- 작업 단위별로 분리하여 요청
- 각 작업 완료 후 검증 단계 포함

## 3. AI 활용 역량

### 3.1 강점 ✅

#### 다양한 작업 유형 활용
- 코드 생성
- 파일 구조 설계
- 대량 파일 생성
- 코드 정리/리팩토링
- 문서 작성

#### AI의 능력 범위 이해
- 대량 반복 작업에 AI 적극 활용
- 복잡한 로직도 자연어로 요청

### 3.2 개선 필요 영역 ⚠️

#### 프롬프트 반복 비효율
**관찰된 패턴**:
```
요청 1: "게시물 연결해줘" → 일부만 완료
요청 2: "아직 안 되는 게 있어" → 추가 수정
요청 3: "이것도 안 돼" → 또 추가 수정
```

**문제점**:
- 같은 문제로 3회 이상 대화 반복
- 초기에 전체 범위를 명확히 했다면 1회로 해결 가능

#### 결과물 검증 프로세스 부재
- AI가 "완료했습니다"라고 하면 그대로 수용
- 생성된 파일의 실제 동작 여부 미확인
- 엣지 케이스 테스트 없음

## 4. 기술적 이해도

### 4.1 강점 ✅

#### 프로젝트 구조 파악
- 페이지 간 관계 이해
- 파일 경로 체계 인지

#### 기본 웹 기술 이해
- HTML 구조 (section, article, div 등)
- CSS 클래스 개념
- JavaScript onclick 이벤트

### 4.2 개선 필요 영역 ⚠️

#### 파일 경로 이해 부족
**문제 상황**: 
링크가 `../postdetail page/index.html`에서 `../postdetail page/detail page/xxx.html`로 변경되어야 함을 직접 파악하지 못함

**권장 학습**:
- 상대 경로 (`../`, `./`) 개념
- 파일 시스템 구조와 URL 매핑

#### 디버깅 역량 부족
**관찰된 현상**:
- "안 돼요"라는 증상만 전달
- 콘솔 에러, 네트워크 탭 확인 없음
- 어떤 파일의 어떤 부분이 문제인지 특정 불가

**권장 학습**:
- 브라우저 개발자 도구 사용법
- 404 에러, JavaScript 에러 읽는 방법

작업 완료 후:
- [ ] 직접 브라우저에서 테스트했는가?
- [ ] 모든 케이스를 확인했는가?
- [ ] 예상과 다른 부분은 없는가?
