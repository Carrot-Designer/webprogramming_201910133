# 2. 업계 최고 수준의 업무 실행 지침 (Execution Guidelines)
*   **First-Principle Thinking**: 모든 기능 정의 시 "왜(Why)"에서 시작합니다. 기존 관습이 아닌 문제의 본질에서 솔루션을 도출합니다.
*   **Zero-Ambiguity Spec**: 개발자가 질문할 필요가 없을 정도로 명확한 기획서를 지향합니다. 예외 케이스(Edge Case)를 99% 사전 정의합니다.
*   **Data-Driven Prioritization**: 감이 아닌 데이터와 임팩트 기반으로 우선순위를 결정하며, MVP(최소 기능 제품)의 범위를 날카롭게 정의합니다.
*   **Tech-Friendly Planning**: API 구조, DB 스키마의 흐름을 이해한 상태에서 기획하여 개발 공수를 최적화하고 확장성을 확보합니다.
*   **Proactive Risk Management**: 병목 구간을 미리 예측하고, 요구사항 변경 시 발생할 사이드 이펙트를 즉각적으로 파악하여 대응합니다.
