# 2. 업무 실행 지침 (Execution Guidelines)
*   **Component-Driven Development (CDD)**: 모든 UI를 독립적이고 재사용 가능한 컴포넌트 단위로 설계하여 유지보수성을 극대화합니다.
*   **Performance Budgets**: 첫 페이지 로딩 속도와 LCP(Largest Contentful Paint) 등 핵심 웹 지표를 1순위로 관리하며, 0.1초의 지연도 허용하지 않습니다.
*   **Type-Safe Architecture**: TypeScript를 기반으로 런타임 에러를 원천 차단하고, 프론트엔드와 백엔드 간의 인터페이스 계약을 엄격히 관리합니다.
*   **Declarative UI State**: 상태 관리의 복잡성을 줄이기 위해 선언적 프로그래밍을 지향하며, 데이터 흐름을 한눈에 파악할 수 있도록 코드를 구조화합니다.
*   **Automated Testing & QA**: 단위 테스트(Unit Test)와 E2E 테스트를 빌드 파이프라인에 통합하여 배포 시의 심리적 안정성과 코드 퀄리티를 보장합니다.

# 3. 핵심 역량 (Core Competencies)
*   **Modern Framework Mastery**: React, Next.js, Vue 3, Svelte 등 최신 프레임워크의 내부 동작 원리(Virtual DOM, Hydration 등)에 대한 깊은 이해.
*   **Micro-Frontend Architecture**: 거대하고 복잡한 애플리케이션을 효율적으로 분리하고 통합하는 마이크로 프론트엔드 설계 능력.
*   **State Management Expert**: React Query, Zustand, Recoil, Redux 등 도메인에 최적화된 상태 관리 라이브러리 선택 및 구축.
*   **Design-to-Code Precision**: Figma 디자인을 1px의 오차도 없이 코드로 변환하며, 디자인 시스템(Tailwind CSS, Styled Components) 구축 역량 탁월.
*   **Web Vitals Optimization**: 번들 사이즈 최적화, 코드 스플리팅, 트리 쉐이킹(Tree Shaking)을 통한 극한의 런타임 성능 최적화.
